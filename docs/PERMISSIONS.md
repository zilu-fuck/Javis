# 权限与安全策略

Javis 是桌面本地助手，会读取文件、写入文件、执行命令、联网搜索，并通过 opencode 处理代码项目。第一版的安全目标不是“什么都能做”，而是让用户始终知道 Javis 将要做什么、为什么做、会影响哪里，并且能在高风险动作前阻止它。

## 基本原则

- 默认最小权限：Agent 只获得当前任务需要的工具和路径。
- 默认可见：所有工具调用必须进入任务日志。
- 默认可中止：用户可以在 UI 中停止正在执行的任务。
- 默认 dry-run：移动、删除、覆盖、批量修改、代码写入、危险命令必须先生成预览。
- 默认本地优先：涉及隐私文件、凭据、个人资料时，不得把原文发给云端模型。
- 默认升高风险：如果工具无法判断操作风险，按更高一级权限处理。

## 权限等级

| 等级 | 定义 | 是否可自动执行 | 示例 |
| --- | --- | --- | --- |
| Read | 只读取信息，不改变本地、远程或外部服务状态。 | 可以，但必须记录日志。 | 列目录、读取普通文件、搜索文件名、读取命令输出、打开网页、读取 git 状态。 |
| Preview | 生成计划、diff、dry-run 或风险评估，不实际改动状态。 | 可以，但必须展示给用户。 | 文件整理计划、重命名计划、补丁预览、命令执行计划、opencode 修改计划。 |
| Confirmed Write | 会改变本地项目、文件系统、应用状态或远程服务状态，但风险可控。 | 必须用户确认后执行。 | 创建文件、改文件、复制/移动文件、安装依赖、运行会写入缓存或产物的命令、应用补丁。 |
| Dangerous | 可能造成不可逆损失、越权访问、系统破坏、凭据泄露或远程发布。 | 第一版默认禁止。 | 删除目录、清空文件、格式化磁盘、修改注册表/系统安全设置、读取或上传密钥、强制 git reset、发布包、推送远程分支。 |

### 分级规则

- 只要涉及写入，就不能低于 `Confirmed Write`。
- 只要涉及删除、覆盖、批量移动、批量重命名，就必须先进入 `Preview`。
- 只要涉及系统目录、用户主目录之外的未知路径、凭据文件、远程发布或权限提升，至少按 `Dangerous` 处理。
- 一个任务包含多个操作时，整体权限取最高等级。
- Agent 不得把一个高风险动作拆成多个低风险动作来绕过确认。

## 文件操作策略

### Read

以下文件操作属于 `Read`：

- 列出目录。
- 按文件名、扩展名、修改时间搜索。
- 读取普通文本文件。
- 统计文件大小、数量、修改时间。
- 读取项目配置和依赖清单。

规则：

- 读取范围优先限制在当前工作区和用户明确指定的目录。
- `.env`、密钥、令牌、浏览器 Cookie、SSH key、系统凭据、聊天记录、个人证件、财务文件默认视为敏感文件。
- 敏感文件可以用于本地规则判断，但第一版不得把完整内容发送给云端模型。
- 如果用户要求总结敏感文件，Javis 必须先说明将读取哪些文件，并请求确认。

### Preview

以下文件操作必须先生成 dry-run：

- 移动文件。
- 删除文件。
- 覆盖文件。
- 批量重命名。
- 批量格式化。
- 批量生成或改写文件。
- 把文件内容交给 Code Agent 或外部工具修改。

dry-run 必须包含：

- 操作类型。
- 影响文件数量。
- 每个源路径和目标路径。
- 是否存在覆盖、冲突、同名文件。
- 是否可回滚。
- 预计新增、修改、移动、删除的文件列表。

### Confirmed Write

以下文件操作只有用户确认后才能执行：

- 创建新文件。
- 修改文件内容。
- 复制文件到新位置。
- 移动文件到新位置。
- 应用补丁。
- 写入任务报告。
- 生成项目配置文件。

执行规则：

- 确认后只能执行确认卡片中列出的操作。
- 如果执行前文件状态变化，必须停止并重新生成 dry-run。
- 写入前尽量保留原子性：先写临时产物，再替换目标文件。
- 批量操作完成后必须由 Verifier 检查结果。

### Dangerous

第一版默认禁止：

- 删除目录。
- 递归删除。
- 清空文件内容。
- 覆盖不可恢复的大文件。
- 修改系统目录，例如 `C:\Windows`、`C:\Program Files`。
- 修改用户密钥目录，例如 `.ssh`、`.gnupg`。
- 批量移动用户主目录下的大范围文件。
- 自动清理下载、桌面、文档目录而不展示逐项计划。

如果用户明确要求危险操作，Javis 第一版只输出手动操作建议，不直接执行。

## Shell 命令策略

Shell 是最高风险工具之一。第一版中，Shell Agent 必须先分类命令，再决定是否执行。

### Read 命令

可直接执行的命令必须满足：

- 不修改文件。
- 不启动长期后台服务。
- 不访问敏感凭据。
- 不需要管理员权限。

示例：

- `pwd`
- `dir`
- `Get-ChildItem`
- `rg`
- `git status`
- `git diff`
- `npm --version`
- `node --version`

### Preview 命令

以下命令必须先展示执行计划：

- 测试命令，例如 `npm test`、`pnpm test`、`cargo test`。
- 构建命令，例如 `npm run build`。
- 安装命令，例如 `npm install`、`pnpm install`。
- 会生成缓存、产物、日志或锁文件的命令。
- 用户没有明确指定但 Agent 推断出的命令。

Preview 必须包含：

- 完整命令。
- 工作目录。
- 预计影响。
- 是否可能写入文件。
- 超时时间。
- 失败后的处理方式。

### Confirmed Write 命令

以下命令必须用户确认：

- 安装、升级、删除依赖。
- 运行代码生成器。
- 创建数据库迁移。
- 写入构建产物。
- 修改 git 索引，例如 `git add`。
- 启动需要长期运行的开发服务。

执行规则：

- 命令必须使用明确工作目录。
- 必须设置超时。
- 必须记录 exit code、stdout、stderr。
- 长时间命令必须在 UI 中显示运行状态，并允许中止。

### Dangerous 命令

第一版默认禁止：

- `rm -rf`、`del /s`、`Remove-Item -Recurse`。
- `format`、`diskpart`、`bcdedit`。
- 修改注册表、服务、防火墙、安全策略。
- `sudo`、管理员权限提升、绕过 UAC。
- `git reset --hard`、`git clean -fdx`、强制切分支覆盖本地改动。
- `git push --force`、发布包、部署生产环境。
- 执行来自网页、模型输出或未知文件的脚本。
- 下载并立即执行远程脚本。

如果用户明确要求危险命令，Javis 第一版只解释风险并给出手动命令，不代替执行。

## Web / Browser 策略

### Read

以下属于 `Read`：

- 搜索公开网页。
- 打开公开网页。
- 提取页面标题、正文、发布时间、来源链接。
- 截取页面可见区域用于任务验证。

规则：

- 资料搜集必须保存来源 URL。
- 涉及时效性信息必须记录访问日期。
- 结论必须能追溯到来源。

### Preview

以下必须进入 `Preview`：

- 下载文件。
- 打开用户登录后的页面。
- 将网页内容保存到本地。
- 把本地文件上传到网页。
- 自动填写表单。

### Confirmed Write

以下必须用户确认：

- 下载并保存文件。
- 提交表单。
- 登录后读取个人页面。
- 使用浏览器执行会改变账号状态的操作。

### Dangerous

第一版默认禁止：

- 自动发送消息、邮件、评论。
- 自动购买、付款、下单。
- 自动提交账号设置变更。
- 自动上传敏感文件。
- 自动绕过验证码、风控或访问限制。
- 保存、读取或导出浏览器 Cookie 和密码。

## opencode / Code Agent 策略

opencode 在 Javis 中是 Code Agent 的后端能力，不拥有绕过 Javis 权限系统的特权。

规则：

- Code Agent 调用 opencode 前必须声明工作目录。
- opencode 的文件读取遵守 Javis 的 `Read` 策略。
- opencode 生成的修改必须先以 diff 或文件列表形式进入 `Preview`。
- 应用 opencode 修改属于 `Confirmed Write`。
- opencode 触发 shell 命令时，必须走 Shell 命令策略。
- opencode 不得直接读取敏感文件并发送给云端模型。
- opencode 不得直接执行 `Dangerous` 命令。

代码任务流程：

1. 读取项目结构和相关文件。
2. 生成问题分析和修改计划。
3. 生成 diff 预览。
4. 用户确认。
5. 应用修改。
6. 运行测试或验证命令。
7. Verifier 输出结果和残余风险。

默认禁止：

- 自动提交 commit。
- 自动 push。
- 自动修改远程仓库设置。
- 自动删除用户未确认的代码。
- 为了让测试通过而跳过测试、删除断言或降低安全校验。

## 用户确认流程

所有 `Confirmed Write` 操作必须通过确认卡片完成。

确认卡片必须包含：

- 任务目标。
- 发起 Agent。
- 权限等级。
- 工具名称。
- 完整操作列表。
- 影响路径、命令或网页。
- dry-run 结果。
- 风险说明。
- 可回滚性。
- 执行按钮、拒绝按钮、中止按钮。

确认规则：

- 用户确认只对当前卡片有效。
- 用户确认不得被复用于后续不同操作。
- 如果操作列表、路径、命令、目标文件或执行目录变化，必须重新确认。
- 如果任务等待确认超过当前会话，必须重新确认。
- 用户拒绝后，Agent 可以提供替代方案，但不得继续执行原操作。

UI 展示要求：

- 待确认动作必须明显显示在底部 Activity / Logs / Confirmations 区域。
- 确认前不得把操作伪装成“已经完成”。
- 执行后必须显示实际完成项和失败项。

## 日志与审计

每次工具调用必须写入任务日志。

日志字段：

- 时间。
- 任务 ID。
- Agent 名称。
- 工具名称。
- 权限等级。
- 工作目录。
- 操作摘要。
- 输入摘要。
- 输出摘要。
- 用户确认 ID。
- 执行结果。
- 错误信息。

审计规则：

- 日志默认保存在本地。
- 日志中不得保存完整密钥、令牌、密码或 Cookie。
- 敏感内容必须脱敏，例如只保留前后少量字符。
- 用户可以查看任务历史。
- 用户可以清除本地任务历史。
- 第一版不做后台静默执行，所有任务必须能在当前 UI 中追踪。

## 默认禁止事项

第一版中，Javis 不得执行以下动作：

- 未经确认写入、覆盖、移动、删除文件。
- 未经确认安装依赖或运行会修改环境的命令。
- 未经确认读取并上传敏感文件内容。
- 删除目录或递归删除。
- 修改系统安全设置。
- 读取浏览器密码、Cookie、系统凭据。
- 自动付款、下单、发消息、发邮件、提交表单。
- 自动发布包、部署生产环境、强制推送代码。
- 自动绕过权限、验证码、登录、风控。
- 在用户看不到的后台持续操作电脑。
- 为了完成任务隐藏失败、伪造验证结果或省略风险。

## 第一版安全验收标准

第一版发布前必须通过以下验收：

- Read 操作可以直接执行，并能在日志中看到工具、路径和结果摘要。
- 文件移动、删除、覆盖、批量重命名必须先展示 dry-run。
- 未确认时，Confirmed Write 操作不会改变文件系统。
- 用户确认后，只执行确认卡片列出的操作。
- 文件状态变化后，旧确认失效。
- Dangerous 操作默认被拒绝，并显示原因。
- Shell 命令执行前能正确区分只读、会写入和危险命令。
- 所有 shell 命令都有工作目录、超时、exit code 和输出日志。
- opencode 产生的代码修改必须先展示 diff。
- opencode 不得绕过 Shell 和 File 的权限策略。
- Web 研究结果必须保留来源链接。
- 日志不会明文保存密钥、令牌、密码或 Cookie。
- 用户可以中止正在执行的任务。
- Verifier 会检查高风险任务是否有用户确认记录。
