## P1-1：JavisError 铺开到 code.rs（~2h）

### 问题

`code.rs`（1317 行）是最大的剩余模块仍用裸 `String` 错误。其内部函数（propose、approve、apply、parse）全部返回 `Result<T, String>`。迁移到 `JavisError` 可以：

- 让调用方区分 Validation vs Io vs Permission 错误
- 与已迁移的 workspace/inspect/audit/mcpserv 保持一致

### 步骤

| 步骤 | 说明 |
|------|------|
| 1 | `code.rs` 内部函数（`parse_proposal`、`apply_patch` 等）返回 `JavisError` |
| 2 | Tauri command 层 `.map_err(|e| e.to_string())` 转换 |
| 3 | 现有 code 相关测试适配新错误类型 |

### 成功标准

- `code.rs` 内部函数全部返回 `JavisError`
- 现有 125 个 Rust 测试全部通过
- `cargo check` 零警告
