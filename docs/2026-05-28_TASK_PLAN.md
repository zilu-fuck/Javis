本地文件扫描 → LLM batch 分类 → 标签 + 归类

```
scanUserDocuments() → FileEntry[] (已有)
    → classifyDocuments(files, provider) → ClassifiedFile[]
    → UI 标签筛选
```

详见 [2026-05-28_MODEL_PROFILE_PLAN.md](2026-05-28_MODEL_PROFILE_PLAN.md)

### 实现

| 步骤 | 文件 |
|------|------|
| `classifyDocuments()` 函数 | `local-knowledge.ts` |
| 分类 prompt 模板 | `packages/core/src/` |
| `fileTool.classifyDocuments` | `app-runtime.ts` |

### 分批策略

- 每批最多 50 个文件（文件名 + 路径 + 扩展名 + 大小）
- `deepseek-v4-flash` context window 128K，50 个文件的元信息约 2-3K tokens
- 超过 50 个文件自动分批，每批独立调用 LLM

### 前置依赖

- P0-1：分类模型需要 secondary slot 的 `capabilities.longContext: true`

### 成功标准

## P0-3：中文 RAG 最小可用（~3h）

### 问题

用户可以在 AI 对话框中引用本地文档。当前需要手动复制粘贴。需要让 AI 能"看到"用户指定的文档内容。

### 方案（L1 最小可用 —— 不做向量库）

```
用户 @mention 一个文件
    → Rust 命令读取该文件内容
    → 注入到 System Prompt / User Message 上下文
    → LLM 基于文件内容回答问题
```

### 实现

| 步骤 | 文件 | 说明 |
|------|------|------|
| `read_file_chunk` Tauri 命令 | `lib.rs` | Rust 端读取文件前 N 行（UTF-8 safe）。与已有 `scan_markdown_documents` 不同：`read_file_chunk` 按路径读取单个文件的文本内容，用于注入 LLM 上下文 |
| `readFileChunk(path)` 封装 | `local-knowledge.ts` | 前端调用 Tauri 命令 |
| Chat 输入 `@` 触发文件选择 | `ChatComposer.tsx` | 弹出文件列表，数据源来自 `scanUserDocuments()` 的已有结果（`FileEntry[]`），按最近修改时间排序 |
| `injectDocumentContext()` | `packages/core/src/` | 将文档内容注入 prompt |
| 单文件对话 | `app-runtime.ts` | 引用文档的 research 模式 |

### 不做

- ❌ 多文件索引
- ❌ 全文检索 / Embedding / 向量搜索
- ❌ 自动分 chunk
- ❌ Rerank

### 成功标准

- 输入 `@` 能选择最近扫描的文档
- 选中后文档内容注入对话上下文
- AI 能基于文档内容回答

## P1-1：ProviderAdapter 模式（~2h）

### 问题

当前 `model-provider.ts` 只支持 OpenAI-compatible 端点。引入 Proma 式的 `ProviderAdapter` 抽象，统一多 Provider：

```
interface ProviderAdapter {
  readonly id: string;
  complete(prompt: string, options?: CompletionOptions): Promise<CompletionResult>;
  stream(prompt: string, options?: StreamOptions): AsyncIterable<CompletionChunk>;
}
```

> `stream` 返回 `AsyncIterable` 而非 `AsyncGenerator`——当前实现走 Tauri events
> （`stream_model_prompt_start` → `listen("stream-model-chunk", ...)` → `listen("stream-model-done", ...)`），
> 底层可以是 SSE / Anthropic SSE / 本地模型，adapter 封装即可。

### 涉及

| 项 | 说明 |
|---|---|
| `ProviderAdapter` 接口 | 定义统一 complete/stream 签名 |
| `OpenAICompatibleAdapter` | 当前逻辑封装进去 |
| `AnthropicAdapter` | 后续 Anthropic 兼容端点 |
| Provider 注册表 | `providerRegistry: Map<string, ProviderAdapter>` |
| Model Profile 集成 | profile 选择 provider → 对应的 adapter |

### 成功标准

- 切换 provider 不需要改调用方代码
- 单元测试覆盖 adapter 分发
